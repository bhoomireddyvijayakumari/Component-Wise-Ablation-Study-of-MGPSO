# MGPSO Component-Wise Ablation Study

A codebase for running and analyzing a component-wise ablation study of Multi-Guide Particle Swarm Optimization (MGPSO). Three architectural components — **Polynomial Mutation (PM)**, **Archive Tournament Selection (TS)**, and **Ring Neighborhood Topology (RING)** — are toggled on/off across eight configurations, each benchmarked on the ZDT and WFG multi-objective test suites over 30 independent runs, with IGD and HV computed for every run.

## Configurations

Each configuration is a self-contained Jupyter notebook implementing a specific combination of the three component flags:

| Notebook | PM | TS | RING |
|---|:---:|:---:|:---:|
| `proposed.ipynb`  | ✅ | ✅ | ✅ |
| `baselinr.ipynb`  | ❌ | ❌ | ❌ |
| `no pm.ipynb`     | ❌ | ✅ | ✅ |
| `no ts.ipynb`     | ✅ | ❌ | ✅ |
| `no ring.ipynb`   | ✅ | ✅ | ❌ |
| `only pm.ipynb`   | ✅ | ❌ | ❌ |
| `only ts.ipynb`   | ❌ | ✅ | ❌ |
| `only ring.ipynb` | ❌ | ❌ | ✅ |

Each notebook is runnable independently and writes its outputs to the corresponding folder under `results/`.

All eight configuration folders under `results/` share the identical `main/` layout, so any two configurations can be compared directly problem-by-problem.

## Requirements

- Python 3.x
- [pymoo](https://pymoo.org/)
- NumPy
- pandas
- matplotlib
- Jupyter

Install dependencies with:

```bash
pip install pymoo numpy pandas matplotlib jupyter
```

## Usage

1. Open the notebook corresponding to the configuration you want to run (e.g. `proposed.ipynb`).
2. Run all cells. Each notebook executes 30 independent runs across the full ZDT + WFG benchmark set.
3. Outputs (metrics CSVs, pivot tables, summary stats, Pareto front plots) are written to `results/<CONFIG_NAME>/main/`.
4. To compare configurations, use the `main_pivot_IGD.csv` / `main_pivot_HV.csv` files from each configuration's folder, or the raw per-problem `*_metrics.csv` files for run-level analysis.

## Benchmark Suite

- **ZDT:** ZDT1, ZDT2, ZDT3, ZDT4, ZDT6
- **WFG:** WFG1–WFG9

## Metrics

- **IGD** — Inverted Generational Distance (lower is better)
- **HV** — Hypervolume (higher is better)

Both are computed per run and aggregated as mean ± standard deviation over 30 runs per problem per configuration.

## Reproducibility

Each configuration's parameters, random seeds, and evaluation settings are fixed within its notebook so results can be regenerated by re-running the notebook end-to-end. Raw per-run metric logs are preserved in `results/` for independent verification.
